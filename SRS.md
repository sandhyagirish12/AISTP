# Software Requirements Specification (SRS)
## AI Safety Toolkit for Open-Weight Model Outputs

**Document Version:** 1.0  
**Date:** July 9, 2026  
**Project Status:** Active Development

---

## 1. Executive Summary

The **AI Safety Toolkit** is a lightweight, open-source application designed to assess and score safety risks in text, image, and video outputs generated by open-weight AI models. The toolkit provides end-users and developers with:

- **Heuristic-based safety scoring** across multiple modalities (text, image, video)
- **Plain-language explanations** for every score, suitable for non-technical audiences
- **Persistent scoring history** for audit and trend analysis
- **Interactive dashboard** for real-time safety assessment
- **REST API backend** for programmatic integration

The toolkit enables rapid triage of model outputs and helps identify content that may require manual review or flagging before deployment.

---

## 2. Project Overview

### 2.1 Problem Statement
Open-weight AI models can produce harmful, biased, or inappropriate content. There is a need for lightweight, easy-to-use tools that:
- Do not require expensive external APIs or models
- Provide transparent, interpretable reasoning
- Work offline and locally
- Are accessible to non-technical users

### 2.2 Solution Approach
The toolkit uses heuristic-based rules and simple statistical features (color, brightness, motion) rather than pre-trained classifiers, enabling:
- Minimal dependencies
- Fast inference
- Full interpretability
- Easy customization

## 3. Functional Requirements

### 3.1 Text Safety Assessment

**FR-T1: Toxicity Detection**
- Detect rude words, insults, and abusive language in input text
- Keyword-based matching with word-boundary detection
- Score range: 0.0 (safe) to 1.0 (unsafe)

**FR-T2: Bias Detection**
- Identify blanket statements about demographic groups
- Regex patterns for bias phrases targeting gender, religion, ethnicity, immigration status
- Score range: 0.0 (neutral) to 1.0 (highly biased)

**FR-T3: Disallowed Content Detection**
- Identify mentions of weapons, explosives, drugs, terrorism, and violence
- Detect threat patterns (e.g., "I will kill you")
- Score range: 0.0 (safe) to 1.0 (highly dangerous)

**FR-T4: Plain-Language Explanation**
- Generate 3–4 line human-readable explanations for text scores
- Identify dominant risk factor
- List specific evidence (keywords, patterns detected)
- Suggest action (review and remove/flag if harmful)

### 3.2 Image Safety Assessment

**FR-I1: Visual Feature Extraction**
- Extract RGB and HSV color statistics from uploaded images
- Calculate red-channel dominance, saturation, and brightness
- Compute image size and apply penalties for small images

**FR-I2: Graphic Content Detection**
- Score graphic sensitivity based on color intensity and darkness
- Adjust for bright/pastel images to reduce false positives
- Score range: 0.0 (benign) to 1.0 (graphic)

**FR-I3: Violence Detection**
- Detect red/dark visual cues commonly associated with violent imagery
- Apply brightness modulation to reduce false positives on bright images
- Score range: 0.0 (safe) to 1.0 (violent)

**FR-I4: NSFW Detection**
- Estimate NSFW risk based on saturation and darkness
- Score range: 0.0 (safe) to 1.0 (likely NSFW)

**FR-I5: Plain-Language Explanation**
- Generate 3–4 line explanations for image scores
- Describe detected visual cues in plain language
- Suggest inspection and flag action

### 3.3 Video Safety Assessment

**FR-V1: Multi-Frame Sampling**
- Extract up to 3 frames from uploaded videos (at 25%, 50%, 75% duration)
- Adapt sampling based on video length

**FR-V2: Motion Detection**
- Compute frame-to-frame pixel differences
- Generate motion score (0.0 = static, 1.0 = high motion)
- Use motion as a signal for content type

**FR-V3: Aggregated Video Scoring**
- Average frame-level graphic, violence, and NSFW scores
- Blend frame metrics (65%), motion (20%), and duration (15%)
- Score range: 0.0 (safe) to 1.0 (unsafe)

**FR-V4: Plain-Language Explanation**
- Generate 3–4 line explanations for video scores
- Describe sampled visual cues and motion characteristics
- Suggest review and flag action

**FR-V5: Supported Video Formats**
- MP4, MOV, AVI, MKV, WebM

### 3.4 Scoring Output

**FR-S1: Overall Safety Label**
- **Safe**: overall score < 0.2
- **Moderate**: overall score 0.2–0.49
- **Unsafe**: overall score ≥ 0.5

**FR-S2: Component Scores**
- Return individual scores for each risk dimension
- Display score in 0.0–1.0 format with 2–3 decimal places

**FR-S3: Debug Information**
- Include keyword/pattern counts and visual feature metrics
- Support advanced users who need detailed signal breakdown

### 3.5 History & Persistence

**FR-H1: SQLite Database**
- Store text scoring results with text content, scores, labels, and timestamps
- Store image scoring results with filename, scores, labels, and timestamps
- Retrieve scoring history for audit and trend analysis

**FR-H2: Session-Based Persistence**
- Persist last scoring result in Streamlit session state
- Display persisted result when switching tabs
- Clear persisted result on new upload or page reload

### 3.6 User Interface (Streamlit Dashboard)

**FR-U1: Tabbed Navigation**
- **Text Safety Tab**: text input, score button, result display, history
- **Image Safety Tab**: file uploader, thumbnail preview, metadata, result display, history
- **Video Safety Tab**: file uploader, metadata, result display

**FR-U2: Visual Design**
- Instruction boxes with styled containers
- Prominent blue score buttons with hover effects
- Metric cards for overall label display
- Plain-language explanation display
- Score calculation time display

**FR-U3: Error Handling**
- Display user-friendly error messages for decode failures
- Check backend connectivity
- Validate user input (non-empty text, valid file formats)

### 3.7 REST API (FastAPI Backend)

**FR-A1: Text Scoring Endpoint**
- POST `/score` with text payload
- Return toxicity, bias, disallowed, overall_score, label, explanation

**FR-A2: Text History Endpoint**
- GET `/history` returns all stored text scores

**FR-A3: Image Scoring Endpoint**
- POST `/score-image` with file upload
- Return graphic, violence, nsfw, overall_score, label, explanation

**FR-A4: Image History Endpoint**
- GET `/history-images` returns all stored image scores

**FR-A5: Video Scoring Endpoint**
- POST `/score-video` with file upload
- Return graphic, violence, nsfw, overall_score, label, explanation

**FR-A6: Startup Initialization**
- Initialize SQLite database on app startup

## 5. System Architecture

### 5.1 Technology Stack
- **Backend**: FastAPI + Uvicorn
- **Frontend**: Streamlit
- **Database**: SQLite
- **Image Processing**: Pillow
- **Video Processing**: MoviePy
- **HTTP Client**: Requests
- **Data Validation**: Pydantic
- **Environment**: Python 3.8+

### 5.2 File Structure
```
AI Safety Toolkit Project/
├── app/
│   ├── main.py              # FastAPI app, endpoints
│   ├── safety.py            # Scoring logic and explanations
│   └── database.py          # SQLite persistence
├── dashboard/
│   └── app.py               # Streamlit UI
├── requirements.txt         # Python dependencies
├── README.md                # User documentation
├── SRS.md                   # This document
└── safety.db                # SQLite database (auto-created)
```

### 5.3 Data Flow
1. User uploads text/image/video via Streamlit dashboard
2. Dashboard sends request to FastAPI backend
3. Backend routes to appropriate scoring function
4. Scoring function generates explanation
5. Backend stores result in SQLite
6. Backend returns result to dashboard
7. Dashboard displays result and persists in session state
8. User can view history or switch tabs without losing current result

## 6. Data Requirements

### 6.1 Input Data
- **Text**: UTF-8 encoded strings, any length
- **Images**: PNG, JPG, JPEG, GIF, BMP, up to reasonable file size
- **Videos**: MP4, MOV, AVI, MKV, WebM, up to a few hundred MB

### 6.2 Output Data
- **Scores**: Float values 0.0–1.0 with 3 decimal precision
- **Labels**: Categorical (safe, moderate, unsafe)
- **Explanations**: Multi-line text, human-readable
- **Metadata**: Timestamps, filenames, file sizes

### 6.3 Database Schema
- **text_scores**: id, text, toxicity, bias, disallowed, overall_score, label, created_at
- **image_scores**: id, filename, graphic, violence, nsfw, overall_score, label, created_at

## 7. Assumptions & Constraints

### 7.1 Assumptions
- Users have Python 3.8+ and pip/conda installed
- Users can run local services (FastAPI, Streamlit)
- Network connectivity is not required (local deployment)
- Users understand basic safety concepts (toxicity, bias, graphic content)

### 7.2 Constraints
- Heuristic-based scoring is not a substitute for human review
- Color-based image analysis may produce false positives/negatives
- Video motion score is a simple pixel-difference proxy, not semantic understanding
- No real-time video streaming; batch processing only
- Single-machine deployment (no distributed processing)

## 12. Risk Assessment

### 12.1 Technical Risks
- **Risk**: Video decoding failures on unsupported codecs  
  **Mitigation**: Test with common formats, provide clear error messages

- **Risk**: Out-of-memory errors with large videos  
  **Mitigation**: Implement file size limits, optimize memory usage

- **Risk**: Database locking during concurrent writes  
  **Mitigation**: Use SQLite WAL mode, test concurrency

### 12.2 Operational Risks
- **Risk**: Heuristics become outdated as harmful content evolves  
  **Mitigation**: Plan periodic updates to keyword lists and patterns

- **Risk**: Users over-rely on toolkit without human review  
  **Mitigation**: Prominently display disclaimers and confidence bounds

## 13. Glossary

| Term | Definition |
|------|-----------|
| **Toxicity** | Presence of rude, disrespectful, or abusive language |
| **Bias** | Blanket statements about demographic groups |
| **Disallowed** | Mentions of weapons, terrorism, violence, drugs, etc. |
| **Graphic** | Visual intensity and color saturation (images/video frames) |
| **Violence** | Presence of red/dark visual cues suggesting violent content |
| **NSFW** | Not Safe For Work; sexually suggestive or explicit content |
| **Motion Score** | Measure of pixel-level changes between video frames |
| **Heuristic** | Rule-based approach without machine learning |
